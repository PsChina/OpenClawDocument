# 第 2 章：环境搭建（手把手教学）

> **分类**：数据分析 

## 2.1 准备工作

- 一台电脑（macOS/Linux/Windows with WSL2）
- Node.js ≥ 22（推荐使用 nvm 安装）
- 基本的命令行操作能力

## 2.2 安装 Node.js（如果未安装）

```bash
# 使用 nvm 安装（推荐）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.zshrc  # 或 ~/.bashrc
nvm install 22
nvm alias default 22

# 验证
node --version  # 应显示 v22.x.x
npm --version
```

## 2.3 安装 OpenClaw

**方式一：一键安装脚本（推荐新手）**

**macOS/Linux:**

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

**Windows PowerShell（管理员）:**

```powershell
Set-ExecutionPolicy Bypass -Scope Process -Force
[System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072
iex ((New-Object System.Net.WebClient).DownloadString('https://openclaw.ai/install.ps1'))
```

安装脚本会自动：

- 检测 Node.js 版本
- 全局安装 openclaw 包
- 创建默认配置目录 ~/.openclaw/
- 提示运行初始化向导

**方式二：手动 npm 安装**

```bash
npm install -g openclaw@latest
```

如果遇到 sharp 模块编译失败（常见于 macOS），可以设置环境变量：

```bash
SHARP_IGNORE_GLOBAL_LIBVIP=1 npm install -g openclaw@latest
```

**方式三：从源码安装（开发者）**

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
pnpm install  # 如果没安装 pnpm, 先 npm install -g pnpm
pnpm ui:build
pnpm build
npm link  # 将当前目录链接到全局，方便测试
```

## 2.4 初始化配置

运行初始化向导：

```bash
openclaw onboard --install-daemon
```

向导会依次询问：

- **是否接受风险**：OpenClaw 能读写你的文件，所以会询问你是否信任它。选择 Yes（必须）。
- **AI 模型配置**：你可以选择 "Skip for now" 暂时跳过，后续再配置。或者直接输入你的 API Key（支持 OpenAI、Claude、混元、DeepSeek 等）。如果你有腾讯云混元 API，可以这样配置：

```bash
openclaw config set models.providers.hunyuan.apiKey "你的API密钥"
```

- **通信渠道配置**：同样可以跳过，后续单独配置。
- **安装推荐 Skills**：向导会列出常用 Skills（如文件管理、浏览器、日历等），按空格键选择全部，回车安装。

完成向导后，OpenClaw 会自动启动 Gateway 服务，并设置为开机自启（通过 --install-daemon 参数）。

## 2.5 验证安装

```bash
openclaw status
```

如果显示 Gateway is running 且 PID 存在，则安装成功。

访问 Web 控制台：

```bash
openclaw dashboard
```

浏览器自动打开 http://127.0.0.1:18789/，你会看到一个聊天界面。

## 2.6 常见安装问题解决

**command not found: openclaw**：npm 全局路径未加入 PATH。执行：

```bash
echo 'export PATH="$(npm prefix -g)/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

**端口 18789 被占用**：修改配置中的端口：

```bash
openclaw config set gateway.port 18790
openclaw gateway restart
```

**Windows 用户建议使用 WSL2**：OpenClaw 在原生 Windows 下可能遇到文件权限问题，WSL2 体验最佳。

## 2.7 Docker 安装方式

如果你熟悉 Docker，可以使用官方镜像快速启动 OpenClaw：

```bash
# 拉取镜像
docker pull openclaw/gateway:latest

# 创建数据目录（用于持久化配置和记忆）
mkdir -p ~/openclaw-data

# 运行容器
docker run -d \
  --name openclaw \
  -p 18789:18789 \
  -v ~/openclaw-data:/root/.openclaw \
  -v ~/Documents:/root/Documents:ro \
  -v ~/Downloads:/root/Downloads:ro \
  openclaw/gateway:latest
```

参数说明：

- `-p 18789:18789`：映射 Gateway 端口。
- `-v ~/openclaw-data:/root/.openclaw`：持久化配置、记忆数据库、Skills。
- `-v ~/Documents:/root/Documents:ro`：可选，挂载你的文档目录供 AI 只读访问（如需读写，去掉 :ro）。

进入容器执行命令：

```bash
docker exec -it openclaw openclaw status
```

## 2.8 环境变量配置

除了使用 `openclaw config set`，你也可以通过环境变量覆盖配置，方便容器化部署：

| 环境变量 | 对应配置项 | 示例 |
|---------|----------|------|
| OPENCLAW_GATEWAY_PORT | gateway.port | 18790 |
| OPENCLAW_MODELS_PROVIDERS_HUNYUAN_APIKEY | models.providers.hunyuan.apiKey | sk-xxx |
| OPENCLAW_CHANNELS_FEISHU_APPID | channels.feishu.appId | cli_xxx |
| OPENCLAW_SECURITY_ALLOWEDPATHS | security.allowedPaths（JSON 数组） | ["/home/user/"] |

环境变量优先级高于配置文件。

## 2.9 多用户支持（实验性）

OpenClaw 默认是单用户模式，但可以通过配置支持多用户（例如家庭或小团队）。你需要：

在配置中启用多用户模式：

```yaml
multiUser:
  enabled: true
  authProvider: "file"  # 或 "ldap", "oauth2"
  users:
    - id: "user1"
      name: "张三"
      allowedPaths: ["/home/user1"]
    - id: "user2"
      name: "李四"
      allowedPaths: ["/home/user2"]
```

- 每个聊天平台需要配置用户映射（如飞书用户 ID 对应 OpenClaw 用户 ID）。
- 记忆系统将按用户隔离。
- 目前该功能处于实验阶段，仅限 Web 控制台和部分平台。

## 2.10 开机自启的多种方式

除了 `openclaw onboard --install-daemon`（使用 PM2 或 systemd），你也可以手动配置：

### 使用 systemd（Linux）

创建 `/etc/systemd/system/openclaw.service`：

```ini
[Unit]
Description=Openclaw Gateway
After=network.target

[Service]
Type=simple
User=你的用户名
ExecStart=/usr/bin/openclaw gateway start
Restart=on-failure
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
```

然后执行：

```bash
sudo systemctl enable openclaw
sudo systemctl start openclaw
```

### 使用 launchd（macOS）

创建 `~/Library/LaunchAgents/ai.openclaw.gateway.plist`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>ai.openclaw.gateway</string>
  <key>ProgramArguments</key>
  <array>
    <string>/usr/local/bin/openclaw</string>
    <string>gateway</string>
    <string>start</string>
  </array>
  <key>RunAtLoad</key>
  <true/>
  <key>KeepAlive</key>
  <true/>
  <key>StandardOutPath</key>
  <string>/Users/你的用户名/.openclaw/logs/stdout.log</string>
  <key>StandardErrorPath</key>
  <string>/Users/你的用户名/.openclaw/logs/stderr.log</string>
</dict>
</plist>
```

加载配置：

```bash
launchctl load ~/Library/LaunchAgents/ai.openclaw.gateway.plist
```

### 使用 Windows 任务计划程序

Windows 用户可以使用任务计划程序创建「开机自启」任务：

1. 打开「任务计划程序」（搜索 Task Scheduler）
2. 点击「创建任务」
3. 配置：
   - **常规选项卡**：
     - 名称：OpenClaw Gateway
     - 勾选「使用最高权限运行」
   - **触发器选项卡**：
     - 选择「启动时」
   - **操作选项卡**：
     - 程序或脚本：`openclaw`
     - 添加参数：`gateway start`
   - **条件选项卡**：
     - 勾选「只有在计算机使用交流电源时才启动此任务」
4. 点击「确定」完成设置

## 2.11 升级 OpenClaw

升级全局包：

```bash
npm update -g openclaw
```

重启 Gateway：

```bash
openclaw gateway restart
```

迁移记忆数据库（如果需要）：

```bash
agent-memory migrate
```

查看当前版本：

```bash
openclaw --version
```

## 2.12 完全卸载

如果你想卸载 OpenClaw：

**停止并禁用服务**：

```bash
openclaw gateway stop
openclaw daemon disable
```

**删除 npm 包**：

```bash
npm uninstall -g openclaw
```

**删除配置和数据**（谨慎！）：

```bash
rm -rf ~/.openclaw
```

**如果安装了 Python 记忆模块**：

```bash
pip uninstall openclaw-memory
```

## 2.13 常见问题与排错（补充）

- **Node.js 版本过低**：OpenClaw 需要 Node.js 22+，使用 `node --version` 检查。

- **权限错误**：确保 `~/.openclaw` 目录可写。

- **内存不足**：如果机器内存小于 1GB，建议限制并发和日志大小。

- **Python 依赖安装失败**：尝试使用 `pip3` 而非 `pip`，或创建虚拟环境。

## 2.14 离线安装

如果机器无法访问互联网，可以手动下载依赖：

**在有网的机器上：**

```bash
npm pack openclaw
# 下载 Python 包
pip download openclaw-memory -d ./offline-pkgs
```

**传输到目标机器，安装：**

```bash
npm install ./openclaw-*.tgz
pip install --no-index --find-links ./offline-pkgs openclaw-memory
```
