# 第 4 章：本地文件管理
 
本文讨论 OpenClaw 的本地文件管理相关内容，包括安装文件操作 Skills、常用文件操作示例、目录文件管理技能实现等...

## 4.1 安装文件管理 Skills

OpenClaw 提供一系列文件操作 Skill，建议全部安装：

```bash
openclaw skills install fs-read fs-write fs-search fs-organize fs-convert
```

## 4.2 常用文件操作示例

### 4.2.1 文件搜索

示例：

```
文本
在下载文件夹中搜索所有包含"报告"的 PDF 文件
```

AI 会调用 `fs-search`，返回匹配的文件列表。

### 4.2.2 批量重命名

示例：

```
文本
将照片目录中所有 IMG_开头的文件重命名为 vacation_2025_序号.jpg
```

AI 可以理解复杂的重命名规则，调用 `fs-write` 执行重命名。

### 4.2.3 文件分类整理

示例：

```
文本
把下载文件中主要文件分类到不同的子文件夹
```

AI 会调用 `fs-organize` Skill，自动创建 Images、Documents、Archives 等文件类别并移动文件。

## 4.3 自定义文件管理 Skill 实战

假设你想实现一个智能管理规则：根据文件名中的日期自动归档到年/月文件夹。

创建 `~/.openclaw/skills/date-sorter.js`：

```javascript
const fs = require('fs').promises;
const path = require('path');

module.exports.exports = {
  name: 'date-sorter',
  description: '按文件名中的日期 (YYYY-MM-DD) 归档文件夹',
  params: [
    { sourceDir: { type: 'string', required: true } },
    { pattern: { type: 'string', default: '\\d{4}-\\d{2}-\\d{2}' } } // 日期正则
  ],

  async execute({ log }, { sourceDir, pattern }) {
    const files = await fs.readdir(sourceDir);
    const dateRegex = new RegExp(pattern);
    let moved = 0;

    for (const file of files) {
      const filePath = path.join(sourceDir, file);
      const stat = await fs.stat(filePath);
      if (!stat.isFile()) continue;

      const match = file.match(dateRegex);
      if (match) {
        const dateStr = match[0];
        const [year, month] = dateStr.split('-');
        const targetDir = path.join(sourceDir, year, month);
        await fs.mkdir(targetDir, { recursive: true });
        const targetPath = path.join(targetDir, file);
        await fs.rename(filePath, targetPath);
        moved++;
        log(`移动文件 ${file} 到 ${targetDir}`);
      }
    }

    return { moved };
  }
};
```

安装自定义 Skill：

```bash
openclaw skills install --local ~/.openclaw/skills/date-sorter.js
```

使用：

```
文本
运行 date-sorter，sourceDir="/Users/name/Downloads"
```

## 4.4 安全注意事项

OpenClaw 默认只允许访问用户目录下的文件（如 ~/Users/name/.openclaw），不能访问系统级别目录。你可以在配置中修改允许的路径：

```yaml
security:
  allowedPaths:
    - /Users/name/Documents
    - /Users/name/Downloads
    - /Volumes/ExternalDrive
```
